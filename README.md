# 源代码说明文档

## 1. 文档说明

本文档用于说明“北京交通大学食堂就餐仿真平台”项目的源码组织结构、各子目录作用、各源代码文件功能以及其他辅助文件用途。

项目采用 B/S 架构开发，后端以 Python + Flask + SimPy 为主，前端采用 HTML + CSS + JavaScript，实现食堂就餐过程仿真、动态展示和统计结果可视化。

---

## 2. 项目总体目录结构说明

项目根目录下的主要内容如下：

```text
D:\simulate
├─ app.py
├─ requirements.txt
├─ cafeteria_sim/
├─ templates/
├─ static/
└─ tests/
```

其中：

- `app.py`：Flask 应用入口文件，负责页面路由和接口路由。
- `requirements.txt`：项目依赖列表。
- `README.md`：项目说明、测试说明和运行方式。
- `.gitignore`：Git 忽略文件配置。
- `cafeteria_sim/`：后端仿真核心代码目录。
- `templates/`：前端页面模板目录。
- `static/`：前端静态资源目录，包括样式和脚本。
- `tests/`：自动化测试代码目录。

---

## 3. 各个子目录说明

### 3.1 `cafeteria_sim/` 子目录说明

该目录是系统的后端核心模块目录，主要用于组织仿真系统的配置、数据模型和仿真引擎代码。

目录下文件包括：

- `__init__.py`
- `config.py`
- `models.py`
- `engine.py`

该目录主要负责：

- 接收和组织仿真参数
- 定义学生、窗口、餐桌等数据对象
- 实现学生到达、排队、打饭、等桌、就餐、离开的完整仿真逻辑
- 输出时间线数据、统计摘要和学生明细

### 3.2 `templates/` 子目录说明

该目录用于存放 Flask 渲染的 HTML 模板文件，是系统前端页面的结构层。

目录下文件包括：

- `base.html`
- `index.html`
- `simulation.html`
- `results.html`
- `404.html`

该目录主要负责：

- 定义页面整体结构
- 提供启动页、仿真页、结果页和异常页
- 配合 Flask 的 `render_template()` 完成前后端页面联动

### 3.3 `static/` 子目录说明

该目录用于存放前端静态资源，包括 CSS 样式文件和 JavaScript 脚本文件。

该目录下又分为两个子目录：

#### `static/css/`

用于存放页面样式文件，控制系统页面布局、颜色、按钮、卡片和图表容器外观。

目录下文件包括：

- `style.css`

#### `static/js/`

用于存放前端页面交互脚本，主要负责参数提交、仿真播放和结果渲染。

目录下文件包括：

- `index.js`
- `simulation.js`
- `results.js`

### 3.4 `tests/` 子目录说明

该目录用于存放项目自动化测试代码，主要测试配置模块、仿真引擎和 Flask 接口。

目录下文件包括：

- `__init__.py`
- `test_config.py`
- `test_engine.py`
- `test_app.py`

该目录主要负责：

- 验证参数转换是否正确
- 验证仿真结果结构是否正确
- 验证接口能否正常创建仿真并返回结果

### 3.5 `__pycache__/` 目录说明

项目中出现的 `__pycache__/` 目录是 Python 自动生成的字节码缓存目录，不属于手工编写的业务源码。

该目录的作用是：

- 提高 Python 模块加载速度
- 保存 `.pyc` 编译缓存文件

在课程设计说明中，一般不作为核心源码分析对象。

## 4. 各个源代码文件说明

### 4.1 根目录源码文件说明

#### 4.1.1 `app.py`

文件作用：

- 项目的 Flask 后端入口文件
- 创建 Web 应用实例
- 定义页面路由和 API 路由
- 调用仿真引擎并缓存仿真结果

主要内容：

- `index()`：返回启动页
- `simulation_view(simulation_id)`：返回仿真页面
- `results_view(simulation_id)`：返回结果页面
- `create_simulation()`：接收前端参数并执行仿真
- `get_simulation(simulation_id)`：根据编号返回仿真数据

在系统中的地位：

- 是前后端数据交互的核心连接点
- 是整个系统的启动入口

---

### 4.2 `cafeteria_sim/` 目录下源码文件说明

#### 4.2.1 `cafeteria_sim/__init__.py`

文件作用：

- 标识 `cafeteria_sim` 为一个 Python 包
- 提供该仿真核心包的整体说明

说明：

- 本文件本身业务逻辑较少，主要用于包初始化和文档说明

#### 4.2.2 `cafeteria_sim/config.py`

文件作用：

- 定义仿真配置类 `SimulationConfig`
- 将前端提交的数据转换为后端可直接使用的配置对象

主要内容：

- `SimulationConfig` 数据类
- `from_payload()` 类方法

主要功能：

- 统一管理窗口数、桌数、仿真时长、到达速率、服务时间、就餐时间等参数
- 对前端传来的字符串做类型转换
- 对异常值或非法值做边界保护

在系统中的作用：

- 是“页面参数”到“仿真参数对象”的转换层

#### 4.2.3 `cafeteria_sim/models.py`

文件作用：

- 定义仿真系统中用到的核心数据模型

主要类：

- `Student`：学生对象
- `Window`：窗口对象
- `Table`：餐桌对象
- `SimulationResult`：仿真结果对象

主要功能：

- 统一描述学生状态变化过程
- 保存窗口和餐桌运行状态
- 封装最终输出的仿真结果数据

在系统中的作用：

- 是后端仿真过程中的数据基础结构

#### 4.2.4 `cafeteria_sim/engine.py`

文件作用：

- 实现整个食堂仿真的核心逻辑

主要内容：

- `CafeteriaSimulation` 类
- `run_simulation(config)` 方法

主要功能：

- 初始化仿真环境
- 根据参数生成窗口和餐桌资源
- 按到达速率生成学生
- 为学生分配窗口队列
- 模拟打饭服务过程
- 分配餐桌并模拟就餐过程
- 周期性记录系统状态
- 生成统计摘要和学生明细

关键方法说明：

- `choose_window()`：选择负载较小的窗口
- `occupy_table()`：占用空闲餐桌
- `release_table()`：释放餐桌
- `student_process()`：单个学生的完整生命周期过程
- `arrival_process()`：持续生成学生到达事件
- `snapshot_process()`：记录时间线状态数据
- `build_summary()`：汇总统计信息
- `run()`：启动整次仿真

在系统中的作用：

- 是整个项目最核心的业务处理模块

---

### 4.3 `templates/` 目录下源码文件说明

#### 4.3.1 `templates/base.html`

文件作用：

- 所有页面共用的基础模板文件

主要功能：

- 定义 HTML 基本结构
- 引入全局样式文件
- 引入 Chart.js 图表库
- 提供 Jinja 模板扩展块

在系统中的作用：

- 被其他页面继承，用于统一页面风格和结构

#### 4.3.2 `templates/index.html`

文件作用：

- 系统启动页

主要功能：

- 提供参数输入表单
- 输入窗口数量、桌子数量、仿真时长、到达速率等参数
- 提供“开始仿真”和“退出系统”按钮

在系统中的作用：

- 是用户进入系统后的第一个页面
- 是仿真参数的输入入口

#### 4.3.3 `templates/simulation.html`

文件作用：

- 仿真过程展示页面

主要功能：

- 展示开始、暂停、继续、重置、结束按钮
- 展示实时统计卡片
- 展示窗口排队情况
- 展示餐桌占用情况
- 展示仿真过程折线图

在系统中的作用：

- 用于动态演示食堂运行过程

#### 4.3.4 `templates/results.html`

文件作用：

- 仿真结果展示页面

主要功能：

- 展示汇总统计指标
- 展示趋势图和窗口柱状图
- 展示学生摘要表

在系统中的作用：

- 用于输出仿真完成后的分析结果

#### 4.3.5 `templates/404.html`

文件作用：

- 异常页面

主要功能：

- 在仿真编号不存在时给出提示
- 提供返回首页入口

在系统中的作用：

- 处理异常访问场景，提高系统友好性

### 4.4 `static/js/` 目录下源码文件说明

#### 4.4.1 `static/js/index.js`

文件作用：

- 启动页脚本文件

主要功能：

- 监听启动页表单提交事件
- 读取表单参数
- 将参数转换为 JSON 并发送到后端接口
- 获取仿真编号后跳转到仿真页

在系统中的作用：

- 负责前端启动页与后端仿真接口之间的交互

#### 4.4.2 `static/js/simulation.js`

文件作用：

- 仿真页脚本文件

主要功能：

- 从后端读取仿真时间线数据
- 控制“开始、暂停、继续、重置、结束”按钮行为
- 按时间顺序播放仿真过程
- 动态更新统计卡片、窗口信息、餐桌状态和折线图

在系统中的作用：

- 负责将后端仿真结果变成前端动态可视化过程

#### 4.4.3 `static/js/results.js`

文件作用：

- 结果页脚本文件

主要功能：

- 从后端读取仿真结果
- 渲染摘要统计信息
- 绘制结果折线图和窗口柱状图
- 填充学生明细表

在系统中的作用：

- 负责结果页的数据可视化展示
