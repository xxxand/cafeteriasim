# 北京交通大学食堂就餐仿真平台

本项目是一个基于 Flask + SimPy 的食堂就餐仿真原型系统。

## 目录说明

- `app.py`：Flask 服务入口。
- `cafeteria_sim/`：仿真核心代码。
- `templates/`：页面模板。
- `static/`：样式和前端脚本。
- `tests/`：自动化测试代码。

## 运行思路

1. 在启动页输入窗口数、桌数、仿真时长等参数。
2. 后端执行一次完整仿真，并返回时间线数据。
3. 前端在仿真页按时间线播放动态过程。
4. 用户进入结果页查看汇总指标和图表。

## 测试说明

本项目建议采用“手动测试 + 自动化测试”相结合的方式。

### 手动测试什么部分

以下内容更适合你在浏览器中手动输入和操作：

- 启动页是否可以正常输入参数。
- 点击“开始仿真”后是否能跳转到仿真页。
- 仿真页中的开始、暂停、继续、重置、结束按钮是否正常工作。
- 统计卡片、窗口排队、餐桌状态、折线图是否正常显示。
- 结果页是否能展示摘要统计、趋势图、柱状图和学生摘要表。

### 自动化测试什么部分

以下内容已经可以通过测试代码自动验证：

- 配置参数转换是否正确。
- 非法参数是否会被边界保护逻辑修正。
- 仿真引擎是否返回合法的时间线、统计摘要和学生明细。
- 餐桌占用数量是否始终处于合法范围内。
- Flask 接口是否能正确创建仿真并返回结果。
- 查询不存在仿真编号时是否返回 404。

### 自动化测试文件说明

- `tests/test_config.py`：测试参数转换和边界保护。
- `tests/test_engine.py`：测试仿真引擎和统计结果。
- `tests/test_app.py`：测试 Flask API 接口。

### 如何运行自动化测试

先安装依赖：

```bash
pip install -r requirements.txt
```

再在项目根目录执行：

```bash
python -m unittest discover -s tests
```

如果只想运行某一个测试文件，可以分别执行：

```bash
python -m unittest tests.test_config
python -m unittest tests.test_engine
python -m unittest tests.test_app
```

## 建议的测试分工

为了写课程报告时更清晰，你可以这样描述：

- 手动测试：负责页面交互、按钮功能、页面跳转和图表显示。
- 自动化测试：负责后端函数、仿真逻辑、数据结构和接口返回。

这样既体现了你做了完整测试，也符合该项目“前后端一体 + 仿真逻辑”的特点。
